This document describes the Age Assurance Level 2 Vocabulary, i.e., the vocabulary is used to express an identity proofing event that meets the the U.S. National Institute of Standards criteria for an Identity Assurance Level 2 vetting as listed in NIST Special Publication 800-63a .

Alternate versions of the vocabulary definition exist in Turtle and JSON-LD.

Class definitions

Level2AgeAssuranceCredential

An age assurance credential that meets NIST Identity Assurance Level 2

This Verifiable Credential communicates information about a document check that would be necessary for a system to validate whether or not to accept it as a valid identity proofing event. This information includes:

  • The Document Number of the individual's identity document (e.g. driver's license #)
  • The Issuing Authority (e.g. a specific DMV)
  • The individual's Date of Birth
  • The Expiration Date of the identity document
  • A cryptographic hash of a credential containing the individual's photo
This mechanism is designed to be used with systems that only collect the minimum amount of information required to perform age verification during a transaction. This mechanism is designed to preserve the individual's privacy and therefore does not collect biometric images or non-relevant data from the identity document (but rather cryptographically links to the biometric information in case the check needs to be performed in the future against the same image used during the check). The NIST Identity Assurance Level 2 Age Assurance Verifiable Credential must be from an issuer which is a trusted Issuing Authority (e.g. a particular DMV) represented by a known Decentralized Identifier (DID).

Level2AgeAssuranceCheck

An age assurance check event that meets criteria defined for NIST Identity Assurance Level 2

An event that binds the following pieces of information together while ensuring that the individual presenting the information matches the information provided:

  • The Document Number of the individual's identity document (e.g. driver's license #)
  • The Issuing Authority (e.g. a specific DMV)
  • The individual's Date of Birth
  • The Expiration Date of the identity document
  • A cryptographic hash of a credential containing the individual's photo

Property definitions

The following are property definitions in the ageloa2 namespace:

personalPhotoCredential

A reference to a personal photo

A personal photo credential references a credential containing an image that has been used during an identity proofing process. The object itself is privacy-preserving and only contains an identifier and a cryptographic hash of the credential containing the image. The image associated with the personal photo credential matches against the image associated with the `identityDocument` and the individual that presented both pieces of information during an interactive identity proofing event that was performed either in person or remotely.

The property's value should be a URL, i.e., not a literal.

Domain:
ageloa2:Level2AgeAssuranceCheck
{
  ...
  "personalPhotoCredential": {
    "id": "urn:uuid:a1486392-763d-11ed-8281-6be4ae1436ac",
    // content hash of referenced PersonalPhotoCredential
    "digestMultibase": "zQmdfTbBqBPQ7VNxZEYEj14VmRuZBkqFbiwReogJgS1zR1n"
  }
  ...
}

identityDocument

An identity document

An identity document that was used during an identity proofing process. The object itself is minimizes the identity document fields shared to the minimum necessary for the transaction. The image associated with the identity document is assumed to be matched against the `personalPhotoCredential` as the during an interactive identity proofing event that was performed either in person or remotely.

Domain:
ageloa2:Level2AgeAssuranceCheck
{
  ...
  "identityDocument": {
    "documentIdentifier": "T73-2348923",
    "expirationDate": "2028-04-18",
    "dateOfBirth": "1982-10-25",
    "issuingAuthority": "UA"
  }
  ...
}
